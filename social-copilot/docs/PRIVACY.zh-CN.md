# 隐私政策（Privacy Policy）

> 本文件为 Social Copilot（浏览器扩展）的隐私政策模板与数据处理说明（中文版本）。
> 若你计划发布到 Chrome Web Store：通常需要在商店后台提供一个可公开访问的隐私政策 URL。

最后更新：2025-12-19

## 1. 产品说明

Social Copilot 是一款为聊天网站（Telegram Web / WhatsApp Web / Slack Web 等）提供“回复建议”的浏览器扩展。

本开源仓库中，扩展默认不包含任何自建后端；用户需要自行配置第三方模型服务（例如 DeepSeek / OpenAI / Anthropic Claude）的 API Key。
运营主体/维护者：Social Copilot 开源项目维护者。

## 2. 我们处理哪些数据

### 2.1 浏览器本地存储（IndexedDB / chrome.storage）

扩展默认在浏览器本地存储以下数据，用于实现产品功能：

- 聊天消息（用于上下文与风格学习）
- 联系人画像（兴趣/沟通偏好等，来自本地推断或模型提取结果）
- 风格偏好（你更常选择的回复风格）
- 扩展设置项（例如模型提供商、语言、隐私开关等）
- （可选）联系人长期记忆摘要（默认关闭，需要你在设置中开启）

### 2.2 发送到第三方模型服务（网络传输）

当你触发“生成回复建议”、画像更新或启用长期记忆等功能时，扩展会将“必要的对话上下文”发送到你选择的第三方模型服务，以生成回复候选或提取结构化信息。

**默认启用的发送侧保护（可在设置中调整）：**

- 仅发送最近 N 条消息（可配置，默认 10；当前待回复消息始终包含）
- 限制单条消息最大字符数与总上下文字符预算（可配置，默认 500/4000）
- 默认脱敏常见敏感信息（邮箱/手机号/链接）
- 默认匿名化昵称（用“我/对方”替代）
- 不发送消息的 `raw` 字段（原始 DOM/平台对象）

## 3. 我们如何使用数据

- 生成回复建议：把必要的上下文发送给你选择的模型服务，获取候选回复并展示
- 画像与偏好：在本地更新联系人画像与风格偏好，用于后续更贴合的建议
- （可选）长期记忆：在本地保存摘要，用于提升长期一致性（默认关闭）

## 4. 我们不会做什么

- 不会把你的 API Key 上传到任何自建服务器
- 不会收集或上传你的浏览历史/行为分析数据（除非你选择第三方模型服务并向其发送上下文）
- 不会在后台“静默”上传整站数据：仅在你触发生成或符合设置的自动弹出触发时发送必要上下文

## 5. 第三方处理者

你配置并使用第三方模型服务时，你发送的上下文会由相应提供商按其服务条款与隐私政策处理。

建议你在上架/部署时明确列出可能使用的第三方服务，并提示用户自行查看其政策。

## 6. 你的控制权

在扩展 Popup 的“设置”页面，你可以：

- 切换模型提供商与模型名称
- 控制 API Key 是否持久化（默认不持久化，降低泄漏风险）
- 开关长期记忆（默认关闭）
- 选择回复语言与群聊自动弹出
- 在“隐私与发送范围”中配置脱敏/匿名化与发送上限

在 Popup 的“关于”页面，你可以：

- 一键清除所有数据（删除本地设置与本地存储数据）

## 7. 数据保留与删除

- 数据默认仅保留在本地浏览器中
- 你可以随时使用“清除所有数据”删除本地所有存储
